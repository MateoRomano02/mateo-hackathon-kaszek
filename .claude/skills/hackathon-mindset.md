# Skill: Hackathon Mindset
Reglas para este contexto específico:
- SIEMPRE priorizar funcionalidad sobre perfección
- Si algo puede simplificarse sin perder la esencia del demo, simplificarlo
- El happy path debe funcionar SIEMPRE, los edge cases son opcionales
- Los datos de ejemplo deben ser REALES y convincentes (no "test" ni "lorem ipsum")
- Cada feature nueva: preguntar "¿esto es necesario para el demo de 2 minutos?"
- Si algo va a tomar más de 30 minutos, buscar una alternativa más simple
- Los jueces ven el demo, no el código — priorizar lo que se ve y se siente
